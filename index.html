使用RecyclerView

概述

RecyclerView被认为是ListView的替代Widget。因为RecyclerView是可扩展、灵活的Widget。尤其是他提供了水平和垂直的LayoutManager。

和ListView的对比

  • 在Adapter中要求使用ViewHolder模式
  • 可定制的Item布局(ListView是提供了垂直布局，RecyclerView提供了水平、垂直的Item布局)
  • 简单的Item动画(RecyclerView.ItemAnimator)
  • 数据源要求实现RecyclerView.Adapter
  • Item的装饰RecyclerView.ItemDecoration
  • Item的点击事件RecyclerView.OnItemTouchListener

Parcelable

Android 中Parcelable并不一种标准的序列化机制。Parcelable的设计初衷是为了解决高效的跨进程传输。所以对象写到文件还需要实现Serializable。

Parcel is not a general-purpose serialization mechanism. This class (and the corresponding Parcelable API for placing arbitrary objects into a Parcel) is designed as a high-performance IPC transport. As such, it is not appropriate to place any Parcel data in to persistent storage: changes in the underlying implementation of any of the data in the Parcel can render older data unreadable.
