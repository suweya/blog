使用Hexo在Github上搭建Blog

一直在寻找和纠结在哪里写blog，虽说不怎么写，但总的有一个偶尔可以写写。
曾用过CSDN、OSCHINA、CNBLOG、自己搭建WordPress，不过都不是很满意。

知道后面看到Hexo，感觉很轻巧，很快速几行命令就可以快速创建。
而且Article存储在Github上，每次写完就可上传到远程仓库上，就像写代码一样的写blog，呵呵。

下面就来说说怎么搭建

首先你需要安装以下环境

  1. 安装NodeJs http://www.nodejs.org/download 下载，一路下一步即可
  2. 安装Git http://git-scm.com/download/
  3. 安装Hexo Hexo是一个基于Node.js的静态博客程序
    使用nmp命令即可安装（点击鼠标右键，选择Git bash）
         npm install hexo

接下来就可开始创建你自己的blog了

  • 在本地新建一个文件夹，名字随便取
  • 在该文件夹下点击鼠标邮件，选择Git bash
  • hexo init 初始化，Hexo会自动的在文件夹下创建所需要的文件，是不是很方便
  • hexo install 安装依赖包
  • hexo generate 生成静态网页文件(hexo g命令也可以)
  • hexo server 在本地运行(hexo s 也可以)
  • 至此，你的blog就可以在本地预览一下了，打开浏览器输入localhost:4000就可以看到了

下一步就可以将你的blog放到Github上了

首先你得有一个Github帐号
如果没有可以在https://github.com/注册一个

配置Github

  • 在你的电脑的C:\Users\xxx(你的用户名)检查一下有没有.ssh文件夹
  • 如果没有可以生成一个新的SSH Key

    $ ssh-keygen -t rsa -C “邮件地址@youremail.com”

    Generating public/private rsa key pair.

    Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

    注意1: 此处的邮箱地址，你可以输入自己的邮箱地址；注意2: 此处的「-C」的是大写的「C」
    然后系统会要你输入密码：

    Enter passphrase (empty for no passphrase):<输入加密串>

    Enter same passphrase again:<再次输入加密串>

添加SSH Key到GitHub

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

  • 打开本地C:\Users\xxx(你的用户名).ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
  • 登陆github系统。点击右上角的 Account Settings—->SSH Public keys —-> add another public keys
  • 把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

至此你的SSH Key已经配置完成了
在你的Github上创建一个空的repository
创建完成后会有一个项目的地址

然后在本地的bolg的文件下的_config.yml,修改deploy部分

deploy:

type: github

repo: 你的项目的地址

然后就可以通过hexo deploy命令将blog部署上去了
这个静态的web网站就被部署到了github，检查一下分支是gh-pages。
gh-pages是github为了web项目特别设置的分支。
至此你的项目已经部署在Github上了，blog的地址可以打开repository右边的setting查看一下

绑定域名

下一步可以绑定域名，可以不用备案的域名哦，这一点很好
在github项目下新建一个CNAME文件，在文件里写入你的域名
不过通过hexo deploy命令部署的话，每一次都会删除所有的文件重新生成，自然CNAME文件就会被删除
所以要把CNAME文件放在source文件夹下

这里我在西部数码申请的域名好像不支持主域名绑定所以我只好绑定在子域名上
就是blog.suweya.net
在域名管理里面的域名解析添加一条如下的记录
主机名 ：blog
解析类型 ： A
对应值 ： 192.30.252.153
TTL ： 600

这样就可以用你自己的域名访问你的blog啦
